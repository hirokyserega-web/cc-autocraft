# cc-autocraft

Распределённая система автоматического крафта для **ComputerCraft: Tweaked**.
Центральный компьютер (Core) управляет складом, базой рецептов и очередью задач;
черепахи-воркеры исполняют крафт через сундуки-буферы.

## Возможности
- **Авто-крафт по дереву зависимостей**: система сама крафтит промежуточные
  предметы (например, доски из брёвен, ступени из досок).
- **Учитывает выход рецепта**: если из ингредиентов получается не 1, а 2/4/…
  предметов (ступени, кнопки и т.п.) — количество партий рассчитывается
  автоматически.
- **Запись рецептов через тест-крафт**: выкладываете крафт в сундук-сканер,
  система крафтит один раз, показывает результат и предлагает сохранить рецепт.
- **Сканирование всех складов**: при заказе показывается, сколько каждого
  ингредиента есть на складе и сколько нужно.
- **Резервирование ресурсов**: ингредиенты бронируются на этапе планирования,
  чтобы два заказа не конфликтовали.
- **Восстановление после перезагрузки**: очередь, резервы, база рецептов и
  выбранный сканер сохраняются в файлы.

## Установка
Запустите эту одну команду в Minecraft (требуется HTTP API) на Core-компьютере:
```bash
wget https://raw.githubusercontent.com/hirokyserega-web/cc-autocraft/main/install.lua
install.lua
```
Установщик скачает все файлы и перезагрузит компьютер. После перезагрузки
запустится `main.lua run`.

## Настройка
1. **Core**: подключить к проводной сети сундуки-склады, сундук-сканер и
   монитор. Установить беспроводной модем.
2. **Воркеры (черепахи)**: у каждой черепахи — беспроводной модем и **два
   соседних сундука** (входной и выходной буферы). Запустить
   `worker/worker.lua run` на каждой черепахе.
3. На Core запустить `core/server.lua run` (или `main.lua run`).

> Воркеру нужно ровно 2 соседних сундука-инвентаря. Они автоматически
> назначаются как входной и выходной буферы и исключаются из общего склада.

## Как пользоваться (UI на мониторе)

Интерфейс имеет 4 вкладки: **ГЛАВНАЯ · СКЛАД · РЕЦЕПТЫ · НАСТРОЙКИ**.

### 1. Выбрать сундук-сканер (НАСТРОЙКИ)
Сундук-сканер — отдельный сундук, где выкладывают крафт для записи рецепта.
Он исключается из склада (это нормально — он служит для рецептов). Нажмите
**ВЫБРАТЬ** рядом со свободным сундуком. Выбор сохраняется между перезагрузками.

### 2. Записать рецепт (РЕЦЕПТЫ)
1. Откройте сундук-сканер в Minecraft и положите ингредиенты в **левый блок
   3×3** (как в верстаке) — они автоматически сдвинутся в **центр 3×3**
   (слоты 4,5,6 / 13,14,15 / 22,23,24). Можно класть сразу в центр.
2. На вкладке РЕЦЕПТЫ видна живая предпросмотр-сетка 3×3 и слот результата.
3. Нажмите **ТЕСТ КРАФТА**. Воркер скрафтит один предмет: результат вернётся
   в слот 16 сканера, остатки ингредиентов — на свои места.
4. Появится окно **«Тест крафта успешен!»** — нажмите **СОХРАНИТЬ**, чтобы
   записать рецепт в базу (с правильным количеством выхода).

### 3. Заказать авто-крафт (РЕЦЕПТЫ или СКЛАД)
- Во вкладке РЕЦЕПТЫ нажмите **КРАФТ** у нужного рецепта, либо во вкладке
  СКЛАД нажмите **ЗАКАЗ** у предмета (работает, если для него есть рецепт).
- В окне заказа видно: выход рецепта, сколько будет изготовлено, выбор
  количества (−/+), и список ингредиентов **«есть / нужно»**.
- Строка **«Можно изготовить: ДА/Нельзя: …»** учитывает промежуточные крафты.
- Нажмите **ЗАПУСТИТЬ КРАФТ** — задачи поедут в очередь (вкладка ГЛАВНАЯ).

### 4. Следить за очередью (ГЛАВНАЯ)
Слева — статус воркеров (СВОБОДЕН/КРАФТИТ/ТЕСТ), справа — очередь задач.
Кнопка **СБРОС** очищает очередь и резервы.

## Как это работает
- **Core** сканирует все сунддуки проводной сети (`storage.refresh`) и считает
  доступные предметы. Сундук-сканер и буферы воркеров в склад не входят.
- **Планировщик** (`core/planner.lua`) строит дерево крафта: grouping
  одинаковых ингредиентов, расчёт партий по `output_count`, резервирование
  сырья. Задачи идут «снизу вверх» (сырьё → промежуточные → итог), а в рамках
  одного заказа выполняются строго по порядку, чтобы родитель не запустился
  раньше дочерних.
- **Диспетчер** (`core/dispatcher.lua`) раздаёт готовые задачи свободным
  воркерам, раскладывая ингредиенты по слотам входного буфера с сохранением
  формы крафта.
- **Воркер** (черепаха) забирает ингредиенты в свою сетку 3×3, крафтит,
  выгружает результат в выходной буфер, сообщает Core.

## Требования
- ComputerCraft: Tweaked (тестировалось на CraftOS 1.9+).
- Core: проводная сеть, монитор, беспроводной модем.
- Воркер: черепаха с крафтом, беспроводной модем, 2 соседних сундука.
- Сундук-сканер и буферы — стандартные сундуки (27 слотов, 9×3).
