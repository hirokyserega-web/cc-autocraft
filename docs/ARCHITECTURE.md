# Архитектура распределенного автокрафтера (cc-autocraft)

## Обзор системы
Система построена по принципу "Центральный сервер - Распределенные воркеры".

### Компоненты
1. **Core (Сервер)**:
   - Управляет хранилищем (через проводную сеть).
   - Ведет базу рецептов.
   - Планирует задачи (дерево зависимостей).
   - Резервирует ресурсы.
   - Диспетчеризирует задачи свободным воркерам.
   - Предоставляет UI.
2. **Workers (Черепахи)**:
   - Исполняют конкретные задачи крафта.
   - Общаются с сервером по Rednet.
   - Работают через буферные сундуки (по умолчанию).
3. **Транспорт (Buffer Mode)**:
   - У каждой черепахи есть входной и выходной сундуки.
   - Core кладет ресурсы во входной буфер, забирает результат из выходного.
   - Черепаха забирает ресурсы из входного, крафтит, кладет в выходной.

## Резервирование ресурсов
Чтобы избежать конфликтов при работе нескольких черепах, Core использует глобальный менеджер резервов. 
Предметы бронируются в момент планирования заказа. Если ресурсов не хватает, заказ не стартует.

## Сетевой протокол
Связь через Rednet с использованием JSON-пакетов.
- `protocol`: cc_autocraft
- `version`: 1.0
- Типы сообщений: DISCOVER, CRAFT_REQUEST, STATUS, RESULT, HEARTBEAT.

## Персистентность
Все состояние очереди и резервов сохраняется в файлы на Core (`/data/queue.dat`, `/data/reserves.dat`). Это позволяет системе восстанавливаться после перезагрузки.
